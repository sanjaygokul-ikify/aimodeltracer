# Contributing to AI Model Tracer
## PR Workflow
1. Fork the repository and create a new branch for your feature or bug fix.
2. Implement your changes, following the code style and commit standards.
3. Create a pull request, providing a detailed description of your changes and their rationale.
## Commit Standards
* Use meaningful commit messages, following the GitHub guidelines.
* Include relevant issue numbers or PR links in your commit messages.
## Code Style Rules
* Follow PEP 8 for Python code.
* Use Markdown for documentation and README files.
